### search_actions Usage
The `search_actions` tool allows you to discover the available functionality within this MCP server.
Provide a `query` to search for specific tools or actions. It returns a list of matching tools that can be executed on `.fdx` files, helping you understand how to interact with the script data.

### Scope of editing vs. preservation
The `edit_*` and `new_file` tools operate on paragraphs, element settings, the title page, headers/footers, the six SmartType dictionaries, dual dialogue, and the spell-check ignore words. Every other top-level FinalDraft block — revisions, macros, tag data, actors, cast, page layout, watermarking, and the rest — is now modeled with typed Go structs (see `fdx/fdx.go`) so it round-trips through `read_fdx` → `save_fdx` with structured field access instead of an opaque blob, but there is still **no `edit_*` tool for them**: `get_macro_alias`/`get_macro_alias_list` read the Macros block, `get_revisions`/`get_tag_data`/`get_display_boards` read the Revisions/TagData/DisplayBoards blocks, and the rest are read/save-only — to modify them, edit the file in FinalDraft. A handful of leaf containers that are always empty in both reference documents (CustomOrder, Holders, FilterSets, AltCollection, ListItems, Writers, Images, LockedPages) still round-trip via a raw inner-XML passthrough rather than a fully modeled shape, since FinalDraft has never shown us their real internal structure. Nested `<DualDialogue>` blocks and the top-level `<SpellCheckIgnoreLists>` block are typed and round-trip losslessly (dual dialogue was previously dropped on save).

### Analysis suite
`get_script_stats`, `get_scene_index`, `get_character_appearances`, `get_page_map`, `get_scene_properties`, `get_scene_arc_beats`, and `get_fdx_breakdown` are read-only tools that derive holistic metadata from the loaded document (scene catalog, pagination, character frequency, arc beats) rather than exposing a single FinalDraft block. They share one computation engine (`tools/breakdown.go`), so results are consistent whether queried individually or via the combined `get_fdx_breakdown` report. None of them modify the document or trigger SmartType updates.
